Overview

The server desktop is the command node for a TALON deployment. It unlocks a local SQLCipher database, owns server sync authority, generates client enrollment tokens, manages operator leases and revocation, and exposes every operational page available to clients plus server-only administration pages.

Installation

Install The Server Artifact

Use the server-specific Linux package. The installer reads the artifact role from .talon-artifact-role; do not pass --mode to the installer.

tar -xzf talon-desktop-server-linux.tar.gz
cd talon-desktop-server-linux
bash ./install.sh --yes

What The Installer Creates

  • Bundle directory: $XDG_DATA_HOME/talon/talon-desktop-server-linux or ~/.local/share/talon/talon-desktop-server-linux.
  • Launcher: ~/.local/bin/talon-desktop-server.
  • Desktop entry: talon-desktop-server.desktop named T.A.L.O.N. Server.
  • Default data directory: ~/.talon-server.
  • Default config: ~/.talon-server/talon.ini.
  • Default Reticulum config: ~/.talon-server/reticulum/config.

First Run And Setup

Unlock Or Create The Local Database

Start talon-desktop-server. Enter the local passphrase. On a fresh database, use a passphrase that meets the policy described in the security section.

Configure Reticulum

Network Setup opens only after database unlock. Use a template, import an existing Reticulum config, validate, then save. Server transport should be enabled for server nodes.

Security Model

TALON uses a local encrypted data model and Reticulum network transport. The UI collects operator input; the core session owns unlock, crypto, sync, enrollment, lease, revocation, and document handling.

Local Data

  • New local databases require a passphrase of at least 12 characters and at least three character classes.
  • The passphrase is run through Argon2id using 64 MB memory, time cost 3, parallelism 1, and a 32 byte output key.
  • SQLCipher protects the local database. Additional PyNaCl SecretBox field encryption is used for protected fields and documents.

Network Identity

  • Reticulum identities are stored as encrypted *.identity.enc files derived from the unlocked database key.
  • Identity directories and files are forced toward owner-only permissions, and symlinked identity paths are rejected.
  • The Network Setup dialog is available only after database unlock.

Transport

  • Sync traffic uses Reticulum. The UI reports the transport family as AutoInterface, Yggdrasil, I2P, TCP, LoRa, or Unknown.
  • Network details are redacted from the status badge. Direct TCP gets a privacy warning because it can expose addressing metadata.
  • The server destination is talon.server.

Documents

  • Uploads are capped at 50 MB, filenames are sanitized, script and executable extensions are blocked, MIME checks are performed, images are re-encoded when Pillow is available, and SHA-256 is verified on download.
  • Macro-capable office formats trigger a warning before save. Server mode can upload and delete; clients download synced documents.
  • Server operators can also renew leases, revoke clients, review audit events, and delete server-owned records.
Direct messages: The current chat code creates DM channels, but DM end-to-end encryption is not implemented in the current core. Treat direct messages as server-readable operational messages.

Server Administration

Enrollment

Use Enrollment to generate one-time client tokens. Give the client the full TOKEN:SERVER_HASH value through an out-of-band trusted channel.

Clients And Operators

The Clients page shows enrolled operators, lease state, revocation state, profile details, and skills. The server can edit profiles, renew leases, and revoke operators. The Operators page provides the same operator-management surface for the general operator list.

Audit And Keys

Audit supports exact event filtering and payload review. Keys shows the server hash, Reticulum identity status, operator identity rows, and server-side revoke controls. The visible group-key rotation control is present, but the current core command set does not implement destructive group-key rotation.

Feature Tour

Map

The Map page displays OSM, TOPO, and Satellite base-layer controls with overlays for assets, mission areas and routes, waypoints, assignments, zones, SITREPs, and operator pings. Server operators can send a Ping Location marker directly from the map and can filter visible assets.

SITREPs

SITREPs support severity and status filters, unresolved/located/pending filters, templates, native coordinates, map-picked locations, mission/assignment/asset links, document links, acknowledgement, assignment, close/disposition, and server-only delete. FLASH and FLASH_OVERRIDE audio is opt-in through the Audio checkbox.

Assets

Assets can be created and edited with category, label, description, and coordinates. Categories are person, safe house, cache, rally point, vehicle, and custom. The server can verify or unverify assets and hard-delete assets after confirmation.

Missions

Missions include title, type, priority, lead, organization, timing, requested assets, AO polygon, route, staging and demob points, key locations, objectives, constraints, support notes, and custom resources. Server lifecycle controls include approve, reject, abort, complete, and delete. Approval opens an asset allocation review before activation.

Assignments And Check-Ins

Assignments provide a command-post board for active work, operator status, support and overdue indicators, check-ins, and acknowledgement.

Chat

Chat includes grouped channels, default channels, custom channel creation, direct-message channel creation, urgent messages, grid-reference attachments from the map context, operator/alert panels, and server-only delete controls for eligible channels and messages.

Documents

Server mode can upload, list, download, and delete documents. Upload validation is enforced by core document handling. Documents can also be linked to SITREPs from the SITREP detail workflow.

Troubleshooting

IssueAction
App will not launch from terminal.Run talon-desktop-server --smoke --mode server from the installed bundle or rerun the installer with --smoke-test.
Network setup keeps appearing.Unlock, open Network Setup, validate the TALON Reticulum config, save it, and restart TALON if Reticulum was already running.
Direct TCP warning appears.The active Reticulum method is direct TCP. Use Yggdrasil, I2P, LoRa, or a trusted VPN when address privacy matters.
A client cannot enroll.Confirm the server is running, the client and server Reticulum configs can reach each other, and the client is using an unexpired TOKEN:SERVER_HASH.
Document upload is blocked.Check file size, extension, MIME type, and whether the file is an executable or script type.
