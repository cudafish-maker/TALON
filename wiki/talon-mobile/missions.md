# Mobile Missions

Mobile mission support is planned as client mission visibility and field status
support first.

## Required Workflows

- View assigned/active missions.
- View mission detail, assets, route, AO, linked SITREPs, and channel.
- Create mission requests only if core policy allows client creation in the
  current release.
- Show mission lifecycle state from core.

## Constraints

- Server approval, abort, complete, delete, and admin workflows stay desktop-only
  initially.
- Mission map overlays come from core map read models.
