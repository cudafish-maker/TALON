Incident Follow-up Workflow

Wireframe for making the incident Follow-up needed checkbox invoke accountable work instead of only storing a filter flag. A checked follow-up expands required detail fields and offers a linked assignment handoff.

Approval Target

  • Incident dialog expands when Follow-up needed is checked.
  • Follow-up details capture action, owner, urgency, due time, type, and notes.
  • Saving can create a linked assignment/task from the follow-up details.
  • Incidents page exposes open follow-ups as a first-class work queue.

Implementation Notes For Approval

Proposed behavior: follow-up details are required only when Follow-up needed is checked. Saving with Create linked assignment checked creates the incident first, then creates an assignment linked to the same mission/location and carrying an incident reference in its handoff notes until a dedicated incident-assignment relation is added. Clearing follow-up requires an outcome note and removes the incident from Open Follow-ups.
