SITREP Screen With Popup Composer And Follow-up Workflow

This wireframe keeps the cleaner SITREP screen from the previous mock, removes the always-visible composer, opens New SITREP in a separate dialog, and preserves the follow-up incident workflow.

Approval Target

  • SITREP page becomes a feed/detail workspace instead of a three-panel feed/composer/detail surface.
  • New SITREP opens a modal composer with the current fields.
  • Selected detail keeps follow-up actions, document linking, and incident handoff visible.
  • Follow-up needed in the incident dialog expands accountable follow-up fields and can create a linked assignment.

4. Resulting Follow-up Work Queue

The incident stores accountability and can create work on the Assignments page.
1

Incident Saved

Incident stores follow-up type, owner, urgency, due time, next action, source links, and follow-up status.

2

Assignment Created

If selected, TALON creates a linked assignment using the next action, owner, mission, location, priority, and source incident.

3

Follow-up Tracked

Incidents page exposes Open Follow-ups, overdue indicators, and closeout outcome fields.

Implementation Notes For Approval

Proposed behavior: the SITREP page keeps feed, filters, metrics, selected detail, activity, and document linking. The New SITREP button opens a modal composer containing the current composer controls. The selected SITREP action is renamed to Create Follow-up Incident. A checked incident follow-up expands required fields and can create a linked assignment after the incident is saved.
