// when a device is taken

You practice the burn before you need it.
Or you don't survive long enough to learn why.

Seized device. Encrypted database. Encrypted identity files. Argon2id key derivation. Without the passphrase, it is a brick. No operational picture. No operator list. No readable traffic. No names. They have hardware and nothing else.

Meanwhile — if your people are prepared — the server operator has already moved.

Step one. Database shred. Identity hash cleared — not archived, not suspended. Deleted. Every enrollment token tied to that identity: invalidated immediately.

Step two. Identity burn. Cached identity file overwritten and destroyed. The server will not recognize this identity again. Not this week. Not ever.

Step three. Key rotation. Every remaining operator receives new keys on next sync. The seized device cannot decrypt a single byte of traffic generated after this moment. Not one byte.

An operator who suspects their own device does not wait for certainty. Self-revocation stops all sync, closes every active link, destroys local identity files, clears enrollment records. The device goes dark. The network continues without it. That operator is out — not the network.
